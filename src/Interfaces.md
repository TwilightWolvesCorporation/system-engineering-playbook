## 5.1. Общие стандарты

Все REST API соответствуют спецификации **OpenAPI 3.1.0** и доступны по базовому URL `https://api.dockies.com/v1`. Версионирование осуществляется через путь (`/v1/...`). Клиенты обязаны указывать желаемую версию API в заголовке `Accept: application/json; version=1.0`.

Асинхронные внутренние события следуют спецификации **AsyncAPI 2.6.0** и передаются через брокер сообщений RabbitMQ (протокол AMQP 0.9.1).

Для всех изменяющих операций (например, создание документа) требуется заголовок **Idempotency-Key** (UUID). Повторная отправка того же ключа в течение 24 часов возвращает ранее полученный результат и не приводит к дублированию.

## 5.4. Политика обработки ошибок, повторных попыток и идемпотентности

### 5.4.1. Формат ошибок (RFC 9457)

При возникновении ошибки API возвращает ответ в формате `application/problem+json` со следующей структурой:

| Поле       | Тип    | Описание                                        |
|------------|--------|-------------------------------------------------|
| `type`     | string | URI идентификатор типа ошибки                   |
| `title`    | string | Краткое описание (например, "Validation Error") |
| `status`   | int    | HTTP код состояния                              |
| `detail`   | string | Человеко-читаемое описание                      |
| `instance` | string | URI проблемного запроса                         |

**Пример ответа с ошибкой валидации (400):**
```json
{
  "type": "https://api.dockies.com/errors/validation",
  "title": "Validation Error",
  "status": 400,
  "detail": "Поле 'initiator_email' должно быть корректным email адресом.",
  "instance": "/api/v1/documents"
}
