## 1.1. Описание системы и её границы

**Dockies (Documents and Cookies)** — корпоративная платформа электронного документооборота и согласования, обеспечивающая:
- создание и версионирование документов (заявки, договоры, служебные записки);
- настраиваемые маршруты согласования с контролем сроков;
- юридически значимый аудит и неотказуемость действий;
- интеграцию с корпоративной инфраструктурой (AD, справочники, уведомления, хранилище).

### Что входит в границы системы
- Управление жизненным циклом документа (создание → согласование → подписание → исполнение → архив).
- Разграничение доступа на основе ролей (AD групп).
- Логирование всех действий с документом (кто, когда, что изменил).
- Генерация дашбордов и уведомлений о статусах.
- Настройка маршрутов согласования администратором.

### Что не входит в границы системы (требует внешних систем)
- Долговременное хранение документов (передаётся во внешний архив).
- Квалифицированная электронная подпись (интеграция с криптопровайдером/УЦ).
- Формирование бухгалтерских проводок и платежей (интеграция с ERP/банком).
- Нормативно-справочная информация о штатной структуре (получается из HRIS/AD).

## 1.2. Внешние сущности (окружение)

### Пользователи
| **Сущность**  | 	***Роль в документообороте***                                                 |
|---------------|--------------------------------------------------------------------------------|
| Инициатор     | 	Сотрудник, создающий документ и отслеживающий его прохождение.                |
| Согласующий   | 	Руководитель или менеджер, принимающий решение (подпись/отказ/делегирование). |
| Юрист         | 	Проверяет договоры на соответствие нормам, даёт юридическое заключение.       |
| Бухгалтер     | 	Проверяет реквизиты, связывает документ с бюджетом, инициирует оплату.        |
| Администратор | 	Настраивает маршруты, права доступа, мониторит логи и производительность.     |

### Смежные системы
| **Система**             | **Назначение взаимодействия**                                     |
|-------------------------|-------------------------------------------------------------------|
| Система единого входа   | Аутентификация, получение учётных записей, групп, ролей.          |
| Хранилище документов    | Сохранение и получение файлов и их версий.                        |
| Система уведомлений     | Отправка оповещений о задачах, сроках, результатах.               |
| Справочник оргструктуры | Актуальная иерархия подчинённости, замещения, штатное расписание. |
| Система криптоподписи   | Формирование и проверка КЭП, штампа времени.                      |
| Долговременный архив    | Приём документов на постоянное хранение после закрытия дел.       |
| Бюджетирование          | Получение остатков бюджетов, формирование заявок на оплату.       |

## 1.3. Потоки данных (входящие / исходящие)

@drawio{https://github.com/TwilightWolvesCorporation/system-engineering-playbook/blob/main/src/diagrams/DockiesDiagramsContext.drawio}

### Инициатор

| **Направление** | **Поток данных**                                                                           |
|-----------------|--------------------------------------------------------------------------------------------|
| → Dockies       | Карточка документа с типом (заявка / договор / служебка), реквизитами и вложенными файлами |
| → Dockies       | Запрос текущего статуса (по ID документа)                                                  |
| ← Dockies       | Форма ввода с автозаполнением полей (контрагенты, суммы)                                   |
| ← Dockies       | Дашборд со списком документов и их статусами (с фильтрами)                                 |
| ← Dockies       | Уведомление о результате согласования (согласован / отклонён / отправлен на доработку)     |

### Согласующий

| **Направление** | **Поток данных**                                                         |
|-----------------|--------------------------------------------------------------------------|
| → Dockies       | Решение: согласовать / отклонить / делегировать другому                  |
| → Dockies       | Резолюция (текстовый комментарий, до 4000 символов)                      |
| → Dockies       | Запрос полного контекста документа (версии, история, комментарии юриста) |
| ← Dockies       | Список входящих задач с приоритетом и сроком исполнения                  |
| ← Dockies       | Уведомление о новой задаче (с кратким содержанием)                       |

### Юрист

| **Направление** | **Поток данных**                                                                 |
|-----------------|----------------------------------------------------------------------------------|
| → Dockies       | Юридическое заключение: «соответствует / не соответствует / требуется доработка» |
| → Dockies       | Постраничные комментарии прямо в документе (с привязкой к версии)                |
| ← Dockies       | Сравнение двух версий документа (подсветка различий)                             |
| ← Dockies       | Актуальные шаблоны договоров (из библиотеки)                                     |

### Бухгалтер

| **Направление** | **Поток данных**                                        |
|-----------------|---------------------------------------------------------|
| → Dockies       | Статус проверки реквизитов: корректны / ошибка          |
| → Dockies       | Привязка документа к статье бюджета (код ЦФО)           |
| ← Dockies       | Сформированный счёт на оплату (на основе реквизитов)    |
| ← Dockies       | Связанные документы: договор → акт → счёт (граф связей) |

### Администратор

| **Направление** | **Поток данных**                                                     |
|-----------------|----------------------------------------------------------------------|
| → Dockies       | Конфигурация маршрута: последовательный / параллельный / с условиями |
| → Dockies       | Назначение прав: роль → группа AD → тип документа                    |
| ← Dockies       | Логи аудита: кто, когда, какое действие, с каким документом          |
| ← Dockies       | Метрики производительности: время согласования, загрузка узлов       |

### Система единого входа

| **Направление** | **Поток данных**                                                           |
|-----------------|----------------------------------------------------------------------------|
| → Dockies       | Учётная запись: логин, email, подразделение, manager (прямой руководитель) |
| → Dockies       | Членство в группах безопасности (для ролей)                                |
| ← Dockies       | Запрос аутентификации (токен / пароль)                                     |

### Хранилище документов

| **Направление** | **Поток данных**                                                           |
|-----------------|----------------------------------------------------------------------------|
| → Dockies       | Полученный файл по ID версии (вместе с метаданными: размер, хеш, дата)     |
| ← Dockies       | Команда сохранить новую версию (с указанием автора и комментария к версии) |
| ← Dockies       | Запрос на получение предыдущей версии (для сравнения)                      |

### Система уведомлений

| **Направление** | **Поток данных**                                                                             |
|-----------------|----------------------------------------------------------------------------------------------|
| ← Dockies       | Отправить уведомление: получатель (email / ID в мессенджере), тема, тело, ссылка на документ |

### Справочник организационная структура

| **Направление** | **Поток данных**                                                              |
|-----------------|-------------------------------------------------------------------------------|
| → Dockies       | Актуальная иерархия подразделений и сотрудников (включая временные замещения) |
| → Dockies       | Штатное расписание (должности, лимиты согласования по суммам)                 |

### Система криптоподписи

| **Направление** | **Поток данных**                                       |
|-----------------|--------------------------------------------------------|
| → Dockies       | Подписанный хеш документа + сертификат + штамп времени |
| ← Dockies       | Хеш документа (SHA-256) для подписания                 |

### Долговременный архив

| **Направление** | **Поток данных**                                                 |
|-----------------|------------------------------------------------------------------|
| ← Dockies       | Пакет документов + опись (по истечении срока хранения в Dockies) |

### Бюджетирование

| **Направление** | **Поток данных**                      |
|-----------------|---------------------------------------|
| → Dockies       | Остаток бюджета по статье / коду ЦФО  |
| ← Dockies       | Заявка на оплату (после согласования) |

## 1.4. Ключевые ограничения и допущения

### Ограничения

| **ID** | **Ограничение**                                                                                        | **Обоснование**                                           |
|:------:|:-------------------------------------------------------------------------------------------------------|:----------------------------------------------------------|
|  C01   | Dockies не может самостоятельно генерировать квалифицированную электронную подпись                     | Юридическая значимость требует внешнего УЦ согласно 63-ФЗ |
|  C02   | Документы хранятся в системе в актуальном состоянии не более 5 лет, затем выгружаются во внешний архив | Политика компании и ёмкость оперативного хранилища        |
|  C03   | Все действия с документами должны логироваться с невозможностью удаления логов                         | Требование неотказуемости и внутреннего аудита            |
|  C04   | Время отклика интерфейса для инициатора и согласующего не более 2 секунд (в 95% случаев)               | NFR на производительность                                 |
|  C05   | Одновременно в системе может быть активно не более 10 000 сессий                                       | Ограничение инфраструктуры (на начальном этапе)           |
|  C06   | Резервное копирование конфигурации маршрутов и прав — каждые 24 часа                                   | Требование администратора к восстановлению                |

### Допущения

| **ID** | **Допущение**                                                                                                  | **Ответственный**       |
|:------:|:---------------------------------------------------------------------------------------------------------------|:------------------------|
|  A01   | Система единого входа содержит актуальные записи о руководителях (атрибут `manager`)                           | IT-отдел                |
|  A02   | Система криптоподписи доступна по API 24/7 с надёжностью 99.9%                                                 | Владелец УЦ             |
|  A03   | Внешний шлюз уведомлений поддерживает отправку email и сообщений в корпоративный мессенджер (Telegram / Teams) | Команда интеграции      |
|  A04   | HRIS предоставляет полный дамп оргструктуры ежедневно в 02:00 UTC                                              | Отдел кадров            |
|  A05   | Долговременный архив поддерживает приём пакетов в формате MTOM / SOAP                                          | Архивная служба         |
|  A06   | Все сотрудники имеют уникальные логины в AD, привязанные к email                                               | Политика идентификации  |
|  A07   | Юридическая значимость подтверждается наличием сертифицированного средства КЭП в контуре интеграции            | Юридический департамент |
